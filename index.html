RPA para Input de PDF's (Web Automation)

Automação de sistema web com Selenium e manipulação de dados com Pandas, otimizando a produtividade diária e zerando falhas.

Python Selenium Pandas / Numpy SQL

Contexto e Problema

O processo de input de dados e upload de pdf's em portais web era estritamente manual, consumindo elevada carga horária dos analistas. Devido ao alto volume de demandas e ao tempo de execução manual, a operação sofria com gargalos (backlog) e suscetibilidade a erros humanos, impactando o SLA de entrega.

Processo e Solução

  • RPA com Python & Selenium: Desenvolvimento de um robô utilizando ActionChains para navegação complexa, interações sistêmicas avançadas e upload automatizado no portal web.
  • Tratamento e Qualidade: Uso das bibliotecas Pandas e Numpy para ler, tratar e validar matematicamente os dados antes do input, garantindo a integridade da informação.
  • Rastreabilidade com SQL: Integração com banco de dados para registrar logs, controlando com precisão quais processos já haviam sido concluídos para evitar duplicidades.

Entregáveis e Resultados

A substituição do trabalho manual pela automação transformou a operação:

  • Escalabilidade: Aumento exponencial na capacidade de processamento diário, com o robô assumindo a demanda de forma otimizada e contínua, reduzindo o tempo de execução pela metade.
  • Qualidade e Satisfação: Eliminação completa de erros de digitação, erradicação do backlog sistêmico e redução drástica do retrabalho, elevando o padrão de qualidade das entregas.

Inteligência Sócio-Econômica dos Municípios Brasileiros

Análise Exploratória (EDA) em dados abertos cruzando mais de 5.500 municípios para extrair inteligência demográfica e validar hipóteses estruturais.

Python Pandas / Numpy Estatística Feature Engineering

Contexto e Problema

O desafio consistia em investigar um vasto conjunto de dados públicos do Brasil, abrangendo os mais de 5.500 municípios, para responder a questões estratégicas e descobrir padrões ocultos sobre a distribuição de riqueza, educação, infraestrutura e força agropecuária no país.

Processo e Solução

  • Saneamento e ETL: Utilização da biblioteca Pandas para tratamento de valores nulos, normalização de tipos e cruzamento de múltiplas bases de dados (MERGE) geográficas e demográficas.
  • Engenharia de Variáveis (Feature Engineering): Criação de novas métricas relativas (KPIs) para viabilizar comparações justas entre cidades de portes diferentes, como a densidade de veículos, acesso bancário per capita e o percentual representativo de estrangeiros.
  • Estatística Inferencial: Aplicação de Regressão Linear Simples (OLS via `statsmodels`) para validar correlações entre geração de riqueza (PIB per capita), longevidade e Índice de Desenvolvimento Humano (IDHM).

Entregáveis e Resultados

O estudo comprovou analiticamente que o acúmulo de riqueza (PIB) isolado não garante proporcionalidade no desenvolvimento humano (IDHM), evidenciando o "Paradoxo da Riqueza" em polos industriais e agropecuários. Além disso, a análise estatística formou a base estrutural de dados que posteriormente alimentou algoritmos preditivos e de clusterização em projetos de Ciência de Dados.

Concreto Inteligente: Previsão com Machine Learning

Desenvolvimento de um modelo preditivo com XGBoost e um otimizador de custos baseado em simulações estatísticas para traços de concreto de alta performance.

Python XGBoost & SHAP Monte Carlo Deploy de API

Contexto e Problema

A formulação de concreto de alta performance exige o balanceamento complexo de múltiplos materiais (cimento, escória, cinza volante, aditivos, etc.). O desafio era criar um modelo capaz de prever a resistência à compressão (MPa) baseando-se nas proporções químicas, com o objetivo de reduzir o tempo de testes físicos em laboratório (que demoram até 28 dias) e otimizar o custo financeiro da obra.

Processo e Solução

  • Prevenção de Multicolinearidade: Aplicação de Feature Engineering para criar a variável "Fator Água/Cimento" (Lei de Abrams), blindando o modelo contra falsos R² (R² de vaidade) e dando sentido físico à IA.
  • Modelagem Avançada: Avaliação contra baselines (Regressão Linear e Random Forest) e escolha do XGBoost, submetido a tuning de hiperparâmetros (RandomizedSearchCV) e Validação Cruzada (K-Fold).
  • Explicabilidade da IA (SHAP): Utilização de SHAP values para abrir a "caixa-preta" do algoritmo, comprovando para os engenheiros quais materiais e lógicas a IA prioriza na hora do cálculo.
  • Otimizador de Custos: Desenvolvimento de um algoritmo baseado em simulações de Monte Carlo que testa milhares de combinações para sugerir a receita comercial mais barata que atenda à resistência alvo.

Entregáveis e Resultados

O modelo XGBoost vencedor atingiu uma precisão impressionante com um R² de 0.9229 e um erro médio absoluto (MAE) de apenas 3.01 MPa. Para garantir usabilidade, o projeto não ficou apenas no Jupyter Notebook: foi "produtizado" em um laboratório virtual interativo, conectado a uma API RESTful em Python que calcula predições estatísticas e custos em tempo real.

Motor Sentinela: Detecção de Fraudes em FIDCs (Nuclea)

Em Desenvolvimento

Pipeline inteligente para identificação de anomalias em Fundos de Investimento em Direitos Creditórios, integrando modelos de Machine Learning e heurísticas de negócio para mitigar riscos financeiros.

Python Isolation Forest & LOF Feature Engineering APIs (Bacen/IBGE)

Status do Projeto: Esta solução encontra-se em fase de desenvolvimento e validação técnica, necessitando de revisões estruturais antes da finalização.

Contexto e Desafio

Desenvolvido para o Enterprise Challenge em parceria com a Nuclea, o projeto foca na segurança do ecossistema de FIDCs. O objetivo é automatizar a detecção de padrões suspeitos como "boletos zumbis", "pagamentos fantasmas" e colusão entre partes.

Arquitetura Técnica

  • Enriquecimento Externo: Integração dinâmica com APIs do BACEN (Selic) e IBGE (CNAE) para contextualizar o risco macro e setorial.
  • Feature Engineering: Criação de variáveis de comportamento (Z-Score setorial e por UF) e análise de concentração de beneficiários.
  • Modelagem Híbrida: Uso de Isolation Forest e Local Outlier Factor (LOF) com redução de dimensionalidade (PCA) para identificar anomalias em dados multidimensionais.
  • Explicabilidade: Implementação de lógica para detalhamento textual do motivo das anomalias detectadas pela IA.

Resultados Preliminares

O motor já classifica registros em níveis de criticidade, permitindo a exportação de um MVP enriquecido para auditoria e visualização em dashboards.
