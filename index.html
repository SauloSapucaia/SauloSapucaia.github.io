Estudos de Caso

BI para Otimização de Processos Operacionais

Análise do fluxo de recebimento e finalização de processos para identificar gargalos, monitorar SLAs e melhorar a eficiência da equipe.

Power BI DAX Power Query Modelagem de Dados

Contexto e Problema

A empresa possuía um alto volume de processos de clientes, mas a gestão era reativa, baseada em relatórios manuais e planilhas. Isso gerava uma falta de visibilidade sobre a produtividade da equipe, o cumprimento de prazos (SLAs) e a identificação de gargalos, impactando a eficiência operacional.

Processo e Solução

O projeto foi desenvolvido para criar uma fonte central de verdade e fornecer visibilidade em tempo real. O processo incluiu:

  • ETL com Power Query: Extração de dados de diversas fontes (planilhas e banco de dados SQL). Realizei a limpeza, transformação e enriquecimento dos dados, como a integração de uma API de feriados para calcular dias úteis com precisão.
  • Modelagem de Dados: Construção de um modelo dimensional (Star Schema) robusto, com tabelas fato (processos) e dimensão (clientes, tempo, status), garantindo performance e escalabilidade.
  • Cálculos com DAX: Desenvolvimento de mais de 30 medidas complexas em DAX para calcular KPIs essenciais, como Tempo Médio de Atendimento (TMA), volume de processos recebidos vs. finalizados e percentual de SLAs no prazo.

Entregáveis e Resultados

O entregável final foi um dashboard interativo em Power BI, permitindo que os gestores filtrassem as informações por cliente, período e status do processo. Os principais resultados foram:

  • Insight Acionável 1: O dashboard revelou que o tempo médio de processos para o "Cliente X" era 40% maior que a média geral. Isso levou a uma investigação que identificou uma falha específica no fluxo desse cliente, permitindo uma correção direcionada.
  • Insight Acionável 2: Foi identificado um gargalo recorrente na etapa de "Análise de Documentos", que atrasava 60% dos processos. Com essa informação, a gestão pôde realocar a equipe e investir em treinamento, reduzindo o tempo da etapa em 25% em 3 meses.

Análise Preditiva de Churn com Python

Análise exploratória e desenvolvimento de um modelo de classificação para identificar clientes com alta probabilidade de cancelamento.

Python Pandas Matplotlib Scikit-learn

Contexto e Problema

Uma empresa de serviços por assinatura enfrentava uma taxa de cancelamento (churn) crescente, mas não possuía uma estratégia proativa para reter clientes. O objetivo era entender os principais motivos do churn e identificar os clientes com maior risco de cancelamento.

Processo e Solução

Utilizando Python e suas bibliotecas de análise de dados, o projeto seguiu as seguintes etapas:

  • Análise Exploratória de Dados (EDA): Com Pandas, Matplotlib e Seaborn, explorei um dataset de clientes para entender o perfil dos que cancelaram. Investiguei a correlação entre variáveis como plano, tempo de contrato, uso de serviços e o churn.
  • Pré-processamento e Engenharia de Features: Realizei a limpeza dos dados, tratei valores ausentes e transformei variáveis categóricas em numéricas para preparar o dataset para a modelagem.
  • Desenvolvimento do Modelo: Treinei um modelo de Regressão Logística com Scikit-learn para classificar clientes como "provável churn" ou "não provável churn". O modelo alcançou uma acurácia de 82% na identificação de clientes que cancelariam.

Entregáveis e Resultados

O principal entregável foi um Jupyter Notebook detalhado com todo o código e as análises, além de um relatório resumindo os insights para a área de negócio. Veja o código no GitHub.

  • Insight Acionável 1: A análise revelou que 70% dos clientes que cancelavam tinham contratos mensais. A recomendação foi criar uma campanha de marketing oferecendo um desconto para a migração para o plano anual, visando aumentar a retenção.
  • Insight Acionável 2: O modelo identificou que clientes com baixo uso do serviço nos primeiros 2 meses tinham 5x mais chances de cancelar. Isso levou à criação de um programa de "onboarding" para engajar novos clientes e demonstrar o valor da ferramenta logo no início.
